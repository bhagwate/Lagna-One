    Instructions for Mac Users

    CLICK HERE TO DOWNLOAD .dmg

    Actions to take AFTER downloading (copy/paste and save):

    Step 1. In the Downloads folder (where it is downloaded from LagnaOne's official website https://AstroLagnaOne.online), double click the downloaded LagnaOne filename with .dmg extension. It starts the install.

    Step 2. Drag/move the app file to the Applications folder.

    Step 3. Open Terminal (found in Applications > Utilities). Run the below command (copy/paste):

    xattr -cr /Applications/LagnaOne.app

    Step 4: In the Terminal, run the below command (copy/paste):

    chmod +x /Applications/LagnaOne.app/Contents/Resources/backend/LagnaOneBackend

    Please confirm the LagnaOne.app filename in your Applications folder and correct the filename if required. It should be exactly as it shows in your Applications folder.

    Step 5. Close Terminal and launch the app again. It should run. You can copy it to the Dock if you wish.

    If you don't perform steps 1 to 5, you will get error "LagnaOne is damaged and can't be opened. You should move it to trash" which is a common issue for any 3rd party executable binary app - mostly free or un-notarised applications in Mac.

    Trouble-shooting for errors in Mac:

    If there's any further issues like "Unexpected end of JSON input" or "LagnaOneBackend cannot be opened":

    A) Go to System Settings → Privacy & Security, scroll down a bit then → General (or "Allow applications from" in later Mac updates) and click "Allow from Anywhere" when it shows security warning. Then run LagnaOne.
    Note: In later Mac updates, select "Select anywhere" from the drop-down.

    B) After doing A, if it still gives the error, then Right-click on LagnaOne in Applications folder, select "Open," and confirm any security prompts. Then run LagnaOne.

    C) As a last resort, temporarily disable Gatekeeper. Please be warned that it reduces security access to your file system. To do that open the terminal by Applications > Utilities then execute the following command (copy/paste):

    sudo spctl --master-disable

    The above command temporary reduces security access to your file system so re-enable it later with following command (copy/paste):

    sudo spctl --master-enable

    Instructions for Linux Users

    CLICK HERE TO DOWNLOAD .AppImage

    Please ensure that you have Fuse installed. You can either install some Fuse manager tool or do the following steps to handle Fuse. Linux system requirements for LagnaOne - The FUSE package (libfuse2 or libfuse3) must be installed, because AppImage needs it to mount itself when run.

    For most Debian based systems like Ubuntu, Mint, Zorin etc.:

    sudo apt update sudo apt install libfuse2 chmod +x LagnaOne.AppImage

    Then run LagnaOne on prompt as:

    ./LagnaOne.AppImage

    For most Fedora based systems:

    sudo dnf install fuse fuse-devel

    For RedHat Linux:

    sudo dnf install libfuse

    In older systems: sudo yum update then sudo yum install fuse fuse-libs
    If ./LagnaOne.AppImage says file not found then do ./LagnaOne-2.2.0.AppImage
